Linux 🐧

En este note vas a encontrar comentarios sobre Linux que te van ayudar en el trabajo diario y los comandos mas utilizados para hacer el mejor troubleshooting

💻 Primero lo primero el Arbol de directorios 🌳📁

#️⃣ /    👉 Directorio raiz.

#️⃣ /bin   👉 Binarios de usuario como cp, echo, grep, mv, etc...

#️⃣ /boot    👉 Ejecutables y archivos de arranque.

#️⃣ /dev   👉 Archivos de volúmenes.

#️⃣ /etc   👉 Confirguración de sistema y apps.

#️⃣ /home 👉 Directorio de personal del User.

#️⃣ /lib   👉 Blibliotecas para los binarios.

#️⃣ /media 👉 Directorio de montaje.

#️⃣ /opt  👉 Apps externas.

#️⃣ /proc    👉 Información de procesos.

#️⃣ /root    👉 Directorio del SuperUser.

#️⃣ /sbin    👉 Binarios del Sistema.

#️⃣ /srv   👉 Archivos relativos a servers (web, FTP, etc).

#️⃣ /sys   👉 Archivos virtuales con info de eventos del sistema.

#️⃣ /tmp  👉 Directorio de archivos temporales.

#️⃣ /usr   👉 Archivos de programas y Apps instaladas.

#️⃣ /var   👉 Directorio de archivos variales, logs, registros, etc...


📁 Cambiar de Directorio (cd <path>)

cd /home/tinchoram


📁📄 Listar (ls -[OPTIONS] <path>) ordenando por los ultimos archivos modificados

ls -lrt <path>


🕵️‍♂️📄 Buscar Palabras o Patrones en archivos (grep [opciones] pattern [ARCHIVO]) Tambien se puede concatenar a otros comandos para usarlo como filtro: COMMAND | grep 'palabra'

grep -l 'palabra' *


🚦🚢 Estado de Puertos (netstat -anp | grep <port>) filtrando con 'grep' por el puerto requerido

netstat -anp | grep 8080


🐧⚙ Lista Procesos (ps [-aefls] [-u UID] [-p PID]) Ver que procesos se estan ejecutando

ps aux
